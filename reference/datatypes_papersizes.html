2.1.2. Paper Sizes (layout.datatypes.papersizes)¶

There are a large number of paper sizes defined in this module. The most common are: LETTER, LEGAL, the ISO A series (e.g. A4), and the ISO B series (e.g. B4).

The ISO 269 paper sizes can be accessed via constant names (e.g. A3 or C5), from size zero to ten. These and other sizes outside the range can be accessed using array subscript notation: A[-1]. The individual size names aren’t documented here, see the corresponding series name (i.e. see A for information on A4, A0 and so on).

Common formats in other countries, as well as more unusual sizes, such as Japanese Shiroku papers, are included.

As well as common paper sizes, the module also includes useful sizes for things like printer’s oversizes (e.g. SRA2 – see SRA), craft paper (e.g. LARGE_SCRAPBOOK [12”x12”]), business cards (e.g. ISO_BUSINESS_CARD) and envelopes (e.g. DL).

Not all of the individual sizes are documented here, see the source code for a complete list.

The functions in this module are used to transform the paper sizes, and orient them correctly.

2.1.2.1. Reference¶

This module holds page sizes and various mechanisms for manipulating them.

It is intended to be also useful as a drop-in replacement for the ReportLab lib.pagesizes module, and so all the definitions in that file are duplicated here (although this module is considerably more comprehensive). The A-size paper sizes of ReportLab are slightly different to those here, because here we use the conventional purchase page sizes.

layout.datatypes.papersizes.A¶

The ISO 269 A-series of paper sizes, used as the standard paper size in almost every country of the world (the US being the most noteable exception).

layout.datatypes.papersizes.A3_PLUS¶

A lightly oversized version of A4 paper, used for full bleed printing on some inkjet printers.

layout.datatypes.papersizes.ANSI_A¶

Size A in the US national standard ANSI Y 14.1, a synonym for LETTER. Note that this is a single paper size, not a series of paper sizes, as per the ISO 269 sizes.

layout.datatypes.papersizes.ANSI_B¶

Size B in the US national standard ANSI Y 14.1, a synonym for ELEVEN_BY_SEVENTEEN, and TABLOID. Note that this is a single paper size, not a series of paper sizes, as per the ISO 269 sizes.

layout.datatypes.papersizes.ANSI_C¶

Size C in the US national standard ANSI Y 14.1. Note that this is a single paper size, not a series of paper sizes, as per the ISO 269 sizes.

layout.datatypes.papersizes.ANSI_D¶

Size D in the US national standard ANSI Y 14.1. Note that this is a single paper size, not a series of paper sizes, as per the ISO 269 sizes.

layout.datatypes.papersizes.ANSI_E¶

Size E in the US national standard ANSI Y 14.1. Note that this is a single paper size, not a series of paper sizes, as per the ISO 269 sizes.

layout.datatypes.papersizes.B¶

The ISO 269 B-series of paper sizes.

layout.datatypes.papersizes.C¶

The ISO 269 C-series of sizes, these are most commonly used as standard envelope sizes to fit unfolded paper of the corresponding A-size. So C5 envelopes fit A5 paper, or A4 paper folded in half.

layout.datatypes.papersizes.DL¶

The envelope size that will comfortably fit a THIRD_A4 sheet or, more commonly, an A4 sheet folded into three. This is the most common envelope size for business communications in Europe.

layout.datatypes.papersizes.ELEVENSEVENTEEN¶

Another alias for ELEVEN_BY_SEVENTEEN. This alias is included because it the name used in the ReportLab library for this paper size.

layout.datatypes.papersizes.ELEVEN_BY_SEVENTEEN¶

The larger standard US paper size, exactly the size of two LETTER sheets. This is a more modern synonym for TABLOID), and the ANSI_B US national standard paper size.

class layout.datatypes.papersizes.ISO269Series(initial, initial_number=0)¶

Bases: object

Instantiate this class to provide a set of paper sizes conforming to ISO 269.

ISO 269 specifies tolerances of at least 1mm in page sizes and these are often used to make sure that each page size is an integer number of mm in each direction. So A4 is of width 210mm, although A0 is 841mm wide. This breaks the normal halving rule, but is a widespread standard.

Instances of this class can be used to retrieve paper sizes by using subscript notation: A[5], for example. There is no limit to the large (lower numbered) sizes that can be calculated in this way. Because this class always rounds to the nearest millimeter, very small paper sizes (high numbered) will be meaningless.

Arguments:

initial
The ‘reference’ paper size for this series. This is usually a large size, most commonly the 0-size.
initial_number
The size number of the initial paper size given in the first argument.
layout.datatypes.papersizes.JIS_A¶

Japan doesn’t strictly use the ISO 269 page sizes, it has its own national standard (JIS P 0138). The A-size paper in that standard conforms to the ISO A-size paper (the two standards mandate different tolerances, however). For the purposes of this module, the two are synonymous.

layout.datatypes.papersizes.JIS_B¶

Japan has its own national standard B size paper (JIS P 0138) that follows the ISO 269 page proportions, but has a different reference size. This object represents that series. The Japanese B size is slightly larger than its ISO cousin.

layout.datatypes.papersizes.LEDGER¶

A standard size for old accounding ledgers. This is the same paper size as the ELEVEN_BY_SEVENTEEN, or TABLOID paper, but in landscape orientation.

layout.datatypes.papersizes.LEGAL¶

A taller version of the US Letter paper size.

layout.datatypes.papersizes.LETTER¶

The most common US paper size. At one point variations of this paper size were used around the world, now it is largely confined to the US and its immediate neighbours. It is the size A in the US national standard ANSI Y 14.1, and is therefore a synonym of ANSI_A.

layout.datatypes.papersizes.RA¶

A slightly oversized version of the ISO 269 A-series of papers. This is most commonly used as a printer’s paper size, where the final paper will be cropped down to its corresponding A-series size. It is not part of ISO 269, but follows the same page proportions. Its sister-series SRA is more common in commercial printing.

layout.datatypes.papersizes.SHIROKU_BAN4¶

The JIS P 0138 Shiroku ban size 4, also known as 4x6/4.

Note that the SHIROKU-series don’t use the ISO 269 paper proportions, and so can’t be halved to make the next size down.

layout.datatypes.papersizes.SHIROKU_BAN5¶

The JIS P 0138 Shiroku ban size 5, also known as 4x6/5.

This paper size also appears in a non-standard size of (191mm x 259mm) when the paper is cut from a larger sheet. This ambiguity derives from this paper as a historical paper size, with variations from one paper-maker to the next. The variant size isn’t present in this module.

layout.datatypes.papersizes.SHIROKU_BAN6¶

The JIS P 0138 Shiroku ban size 6, also known as 4x6/6.

layout.datatypes.papersizes.SIS_E5¶

Sweeden uses the ISO 269 paper sizes, but adds additional paper sizes in its national standard SIS 014711. There are only two common sizes that are not ISO 269 specified. This one and SIS_G5.

layout.datatypes.papersizes.SIS_G5¶

Sweeden uses the ISO 269 paper sizes, but adds additional paper sizes in its national standard SIS 014711. There are only two common sizes that are not ISO 269 specified. This one and SIS_E5.

layout.datatypes.papersizes.SRA¶

An oversized version of the ISO 269 A-series of papers used for full-bleed printing. SRA2 is the most common bulk paper size for commercial printing, although smaller full-bleed digital presses use SRA3.

layout.datatypes.papersizes.TABLOID¶

The larger standard US paper size, exactly the size of two LETTER sheets. This is a (more archaic) synonym for ELEVEN_BY_SEVENTEEN, but should be treated as deprecated, since TABLOID can refer to other sizes in other contexts (in the printing of newspapers, for example). It also matches the ANSI_B US national standard paper size.

layout.datatypes.papersizes.THIRD_A4¶

The size of an A4 page folded in three on its long axis. This is a common paper size in European businesses, where it acts as a ‘compliment slip.

layout.datatypes.papersizes.bleed(paper_size, bleed)¶

Adds the given bleed to the given paper size. Standard sizes are 3mm internationally and 1/8” US. Large images and die cuts have a larger bleed.

layout.datatypes.papersizes.calculate_up(paper_size, page_size, page_bleed, separation, margin)¶

Calculates how many pages of the given page size can be fit into the given paper size when using a regular rectangular tiling.

Each page should have the given amount of bleed, and bleeds should be separated by the given amount, and the whole piece of paper should have the given outside margin.

The final result is given as an 3-tuple (x, y, rotated) pair of integers showing how the pages should be laid out. The rotated property is true if the pages should be placed rotated by 90 degrees from the orientation given.

It is possible in some cases to fit more pages on the paper by having different pages oriented in different ways. This method doesn’t try to maximize the tiling in that way, and most printers prefer to have things tiled rectangularly so that a cutter can use a minimal set of dimensions.

layout.datatypes.papersizes.flip(paper_size)¶

Reverses the orientation of the given paper size.

layout.datatypes.papersizes.half(paper_size)¶

The paper that you get from cutting the given paper in half along its long axis. A convenience for subdivide(paper_size, 2, 1)

layout.datatypes.papersizes.half_short(paper_size)¶

The paper that you get from cutting the given paper in half along its short axis. A convenience for subdivide(paper_size, 1, 2)

layout.datatypes.papersizes.is_landscape(paper_size)¶

Checks if the given paper is landscape oriented.

layout.datatypes.papersizes.is_portrait(paper_size)¶

Checks if the given paper is portrait oriented.

layout.datatypes.papersizes.is_square(paper_size)¶

Checks if the given paper is square.

layout.datatypes.papersizes.landscape(paper_size)¶

Ensures the paper is in landscape orientation. If it already is, no change is made.

layout.datatypes.papersizes.large_square(paper_size)¶

Makes a square paper size using the larger dimension of the given paper size.

layout.datatypes.papersizes.portrait(paper_size)¶

Ensures the paper is in portrait orientation. If it already is, no change is made.

layout.datatypes.papersizes.quarter(paper_size)¶

The paper that you get from cutting the given paper in half along both its axes. A convenience for subdivide(paper_size, 2, 2)

layout.datatypes.papersizes.small_square(paper_size)¶

Makes a square paper size using the smaller dimension of the given paper size.

layout.datatypes.papersizes.subdivide(paper_size, long_axis_strips=1, short_axis_strips=1)¶

The paper that you get from cutting the given paper into the given number of equal strips along each of its axes. The resulting paper will be rotated so it is in the same landscape or portrait orientation as its parent (if the input is square and the result is not, the result will be landscape). Unlike the ISO 269 series papers, the resulting size isn’t rounded down, so this can be used to find the A5 you get from cutting A4 in half (for example).
